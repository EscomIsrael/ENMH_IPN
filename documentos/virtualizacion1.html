6.2 Virtualización

La virtualización en sistemas operativos consiste en crear entornos de software simulados (máquinas virtuales) que imitan el hardware de una computadora física, permitiendo ejecutar múltiples sistemas operativos o aplicaciones en un solo dispositivo.

Beneficios de la Virtualización

  • Consolidación de servidores: Reducción de hardware físico
  • Aislamiento: Fallos en una VM no afectan a otras
  • Portabilidad: Migración fácil entre hosts
  • Eficiencia: Mejor utilización de recursos
  • Pruebas: Entornos aislados para desarrollo

Casos de Uso Comunes

  • Centros de datos y computación en la nube
  • Desarrollo y pruebas de software
  • Ejecución de aplicaciones heredadas
  • Seguridad (sandboxing)
  • Escritorios virtuales (VDI)

6.2.1 Emulación

La emulación es una técnica que permite a un sistema informático (el anfitrión) imitar las funciones de otro sistema (el invitado) para que el software diseñado para este último pueda ejecutarse en el primero.

Características Clave

  • Traducción completa de instrucciones en tiempo real
  • Puede emular arquitecturas de hardware diferentes
  • Mayor flexibilidad pero menor rendimiento
  • Útil para sistemas heredados o arquitecturas obsoletas

Ejemplos Populares

  • QEMU: Emulador genérico de máquinas completas
  • DOSBox: Emulador de sistemas DOS
  • MAME: Emulador de hardware arcade
  • Wine: Capa de compatibilidad para aplicaciones Windows en Linux
# Ejemplo: Ejecutar un sistema ARM en x86 usando QEMU
qemu-system-arm -machine virt -cpu cortex-a15 \
    -kernel zImage -initrd rootfs.img \
    -append "console=ttyAMA0" -nographic

Comparación: Emulación vs Virtualización

Característica Emulación Virtualización
Rendimiento Bajo (traducción de instrucciones) Alto (ejecución nativa)
Arquitectura Puede ser diferente (ARM en x86) Debe ser la misma
Uso de recursos Alto Moderado
Caso de uso típico Sistemas heredados, desarrollo cross-platform Servidores, cloud computing

6.2.2 Virtualización asistida por hardware

La virtualización asistida por hardware utiliza características específicas del procesador para mejorar el rendimiento y la seguridad de las máquinas virtuales.

Tecnologías Principales

  • Intel VT-x: Extensiones de virtualización de Intel
  • AMD-V: Tecnología de virtualización de AMD
  • SLAT (Second Level Address Translation): Para mejor manejo de memoria
  • IOMMU: Asignación directa de dispositivos a VMs

Ventajas Clave

  • Rendimiento cercano al nativo
  • Mayor seguridad (aislamiento de hardware)
  • Soporte para sistemas operativos no modificados
  • Reducción de sobrecarga del hipervisor
# Verificar soporte de virtualización en Linux
grep -E '(vmx|svm)' /proc/cpuinfo

# Habilitar módulos KVM
sudo modprobe kvm
sudo modprobe kvm_intel # o kvm_amd para procesadores AMD

# Verificar que KVM esté funcionando
lsmod | grep kvm

Tipos de Hipervisores

Tipo 1 (Bare-metal)

  • Ejecuta directamente sobre el hardware
  • Alto rendimiento
  • Ejemplos: VMware ESXi, Microsoft Hyper-V, Xen

Tipo 2 (Hosted)

  • Ejecuta como aplicación en un SO anfitrión
  • Mayor flexibilidad
  • Ejemplos: VirtualBox, VMware Workstation, QEMU

6.2.3 Paravirtualización

La paravirtualización es una técnica de virtualización donde el sistema operativo invitado es modificado para ser consciente de que está siendo virtualizado, lo que permite un mejor rendimiento.

Características Principales

  • Requiere modificaciones en el kernel del SO invitado
  • Proporciona mejor rendimiento que la virtualización completa
  • Usa llamadas hipervisor (hypercalls) en lugar de instrucciones privilegiadas
  • Menor sobrecarga en operaciones de E/S

Implementaciones Notables

  • Xen: Hipervisor que popularizó la paravirtualización
  • Virtio: Framework para dispositivos virtualizados
  • Hyper-V Enlightenments: Optimizaciones para Windows en Hyper-V
# Ejemplo: Configurar un disco paravirtualizado en KVM
# Usando virtio para mejor rendimiento
<disk type='file' device='disk'>
  <driver name='qemu' type='qcow2' cache='none'/>
  <source file='/path/to/disk.qcow2'/>
  <target dev='vda' bus='virtio'/>
</disk>

Arquitectura Virtio

Virtio es un estándar para dispositivos virtuales que aprovecha la paravirtualización:

  • Frontend: Controlador en el SO invitado
  • Backend: Implementación en el hipervisor
  • Virtqueue: Mecanismo de comunicación

6.2.4 Contenedores

Los contenedores son una forma de virtualización ligera que permite ejecutar múltiples instancias aisladas de aplicaciones en un solo sistema operativo anfitrión.

Características Clave

  • Comparten el kernel del sistema anfitrión
  • Mucho más ligeros que las máquinas virtuales tradicionales
  • Aislamiento mediante namespaces y cgroups
  • Inicio rápido y alta densidad

Tecnologías Principales

  • Docker: Plataforma más popular para contenedores
  • LXC/LXD: Contenedores Linux tradicionales
  • Kubernetes: Orquestador de contenedores
  • Podman: Alternativa a Docker sin daemon
# Ejemplo básico con Docker
# Descargar una imagen de Nginx
docker pull nginx

# Ejecutar un contenedor
docker run -d -p 8080:80 --name my-nginx nginx

# Ver contenedores en ejecución
docker ps

# Detener el contenedor
docker stop my-nginx

Contenedores vs Máquinas Virtuales

Característica Contenedores Máquinas Virtuales
Nivel de aislamiento Proceso (namespace/cgroups) Hardware completo
Tiempo de inicio Segundos Minutos
Consumo de recursos Mínimo Significativo
Sistema operativo Comparte kernel con anfitrión Kernel completo independiente
Portabilidad Alta (misma arquitectura) Moderada (imágenes grandes)

Seguridad en Contenedores

Aunque los contenedores proporcionan aislamiento, requieren configuraciones adicionales para seguridad:

  • Namespaces: Aislan procesos, red, sistema de archivos
  • Control Groups (cgroups): Limitan uso de recursos
  • Capabilities: Permisos granulares para procesos
  • SElinux/AppArmor: Políticas de seguridad obligatorias
  • Imágenes firmadas: Verificación de integridad
# Ejecutar un contenedor con restricciones de seguridad
docker run -d \
  --name secure-app \
  --cap-drop ALL \
  --cap-add NET_BIND_SERVICE \
  --memory 512m \
  --pids-limit 100 \
  --security-opt no-new-privileges \
  nginx
